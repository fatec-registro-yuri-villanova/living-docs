## 🧩 Gerenciamento de Estado

No Jetpack Compose, a UI é **reativa**: ela muda quando o estado muda.

### 1. Estado Interno (Stateful)
Usamos `remember { mutableStateOf(...) }` para que o Compose "lembre" de um valor durante a recomposição. Um componente que gerencia seu próprio estado é chamado de **Stateful**.

### 2. Elevação de Estado (State Hoisting)
Para tornar componentes mais reutilizáveis e fáceis de testar, "elevamos" o estado para o componente pai. O componente filho torna-se **Stateless** (não possui estado interno), recebendo os dados via parâmetro e notificando mudanças via eventos (lambdas).

## 💻 Código de Referência

### Data Class com Nullable

```kotlin title="ui/screens/aula02/Aula02Screen.kt"
// Data class para representar o perfil do usuário
data class UserProfile(
    val name: String,
    val role: String? // Nullable para demonstrar SafeCall ?.
)
```

### State Hoisting na Prática

```kotlin title="ui/screens/aula02/Aula02Screen.kt"
@Composable
fun Aula02Screen(modifier: Modifier = Modifier) {
    // 1. ESTADO INTERNO (Stateful)
    // O estado é gerenciado aqui e "elevado" para o filho
    var activeProfile by remember {
        mutableStateOf(UserProfile("Aluno MergeSkills", "Developer"))
    }

    // Chamada do filho passando dados e o evento de mudança
    UserProfileContent(
        profile = activeProfile,
        onProfileChange = { newProfile -> activeProfile = newProfile },
        modifier = modifier
    )
}

@Composable
fun UserProfileContent(
    profile: UserProfile,
    onProfileChange: (UserProfile) -> Unit,
    modifier: Modifier = Modifier
) {
    // 2. STATELESS COMPOSABLE
    // Apenas exibe o que recebe e avisa quando algo acontece
    Column(
        modifier = modifier
            .fillMaxSize()
            .padding(24.dp)
    ) {
        // ... (Layout da tela)
        
        Button(
            onClick = {
                onProfileChange(UserProfile("Visitante", null))
            }
        ) {
            Text("Simular Nullable (SafeCall)")
        }
    }
}
```

:::tip Vantagem do Hoisting
Componentes **Stateless** (como o `UserProfileContent`) são muito mais fáceis de pré-visualizar no Android Studio e de testar, pois não dependem de uma lógica interna complexa de estado.
:::
