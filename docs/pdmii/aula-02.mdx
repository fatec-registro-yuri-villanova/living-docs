## 🧩 Gerenciamento de Estado

No Jetpack Compose, a UI é **reativa**: ela muda quando o estado muda.

### 1. Estado Interno (Stateful)
Usamos `remember { mutableStateOf(...) }` para que o Compose "lembre" de um valor durante a recomposição. Um componente que gerencia seu próprio estado é chamado de **Stateful**.

### 2. Estado Salvo (rememberSaveable)
O `remember` mantém o estado apenas durante a recomposição. Se a tela for destruída (ex: rotacionar o celular), o estado é perdido. Para persistir dados nessas situações, usamos o **`rememberSaveable`**. Ele salva o estado no `Bundle` do Android automaticamente para tipos primitivos ou objetos que implementam `Parcelable`.

### 3. Elevação de Estado (State Hoisting)
Para tornar componentes mais reutilizáveis e fáceis de testar, "elevamos" o estado para o componente pai. O componente filho torna-se **Stateless** (não possui estado interno), recebendo os dados via parâmetro e notificando mudanças via eventos (lambdas).

### 4. Sobrevivendo a Mudanças (rememberSaveable)
O `remember` mantém o estado apenas durante a recomposição. Se a tela for destruída (ex: rotacionar o celular), o estado é perdido. Para persistir dados nessas situações, usamos o **`rememberSaveable`**.

:::info Kotlin Parcelize
Para usar o `rememberSaveable` com objetos customizados, a classe deve implementar `Parcelable`. Ativamos o plugin `id("kotlin-parcelize")` no `build.gradle.kts` para gerar esse código automaticamente com a anotação `@Parcelize`.
:::

## 💻 Código de Referência

### Data Class Parcelável

```kotlin title="ui/screens/aula02/Aula02Screen.kt"
import android.os.Parcelable
import kotlinx.parcelize.Parcelize

@Parcelize
data class UserProfile(
    val name: String,
    val role: String?
) : Parcelable
```

### State Hoisting com rememberSaveable

```kotlin title="ui/screens/aula02/Aula02Screen.kt"
@Composable
fun Aula02Screen(modifier: Modifier = Modifier) {
    // ESTADO SALVO: O 'rememberSaveable' sobrevive à rotação de tela
    var activeProfile by rememberSaveable {
        mutableStateOf(UserProfile("Aluno MergeSkills", "Developer"))
    }

    UserProfileContent(
        profile = activeProfile,
        onProfileChange = { newProfile -> activeProfile = newProfile },
        modifier = modifier
    )
}

@Composable
fun UserProfileContent(
    profile: UserProfile,
    onProfileChange: (UserProfile) -> Unit,
    modifier: Modifier = Modifier
) {
    // Componente Stateless (Não gerencia estado)
    Column(
        modifier = modifier
            .fillMaxSize()
            .padding(24.dp)
    ) {
        // ... (Layout da tela)
        
        Button(
            onClick = {
                onProfileChange(UserProfile("Visitante", null))
            }
        ) {
            Text("Simular Nullable (SafeCall)")
        }
    }
}
```


:::tip Vantagem do Hoisting
Componentes **Stateless** (como o `UserProfileContent`) são muito mais fáceis de pré-visualizar no Android Studio e de testar, pois não dependem de uma lógica interna complexa de estado.
:::
